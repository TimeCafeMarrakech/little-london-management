# UI Design System

## 1. Design Philosophy

The Little London Management System follows the approved Premium Boutique Dashboard v3 design language.

The interface should feel:

- Premium
- Boutique
- Warm
- Calm
- Child-centred
- Professional
- Modern SaaS
- Trustworthy
- Easy to understand

The system should not feel like a traditional school ERP. It should feel like a premium digital product for a boutique education business.

The Premium Login Experience and Premium Dashboard v3 are the official visual references for all future UI work.

---

## 2. Brand Identity

The Little London brand combines a premium education feel with a warm Play & Learn personality.

Core brand traits:

- British-inspired
- Family-focused
- Friendly
- Elegant
- Clean
- Child-centred but not childish
- Professional but not corporate

Brand usage:

- Use the Little London Play & Learn coral bus logo style for friendly surfaces.
- Use warm, soft UI elements instead of harsh corporate styling.
- Prefer calm hierarchy and generous spacing over dense layouts.

---

## 3. Colour Palette

Use the approved Premium Boutique Dashboard v3 palette.

### Primary Colours

- Coral: `#F24A3A`
- Navy: `#0F2747`
- Sage Green: `#A8C3B0`
- Warm Cream: `#F8F6F2`
- Warm Yellow / Soft Gold: `#D6B36A`

### Supporting Colours

- Soft Mint: `#BFE2D0`
- Light Blue: `#DDEAF5`
- Soft Border: `#EADFCF`
- Muted Text: `#5B6F82`
- Soft Surface: `#FFF8EE`

### Usage

- Navy for primary text.
- Coral for primary actions, active states, and brand emphasis.
- Sage for calm success states and child-centred accents.
- Warm yellow for attention, finance, and secondary highlights.
- Cream for page backgrounds and large surfaces.

Avoid:

- Purple gradients
- Harsh black
- Neon colours
- Dense grey ERP styling
- Bright cartoon palettes

---

## 4. Typography

Typography should be clean, modern, and readable.

Use:

- Strong heading hierarchy
- Navy text for primary headings
- Muted navy/grey for supporting text
- Uppercase micro-labels only for section labels or small badges
- Generous line height for descriptions

Guidelines:

- Page titles should be bold and clear.
- Card headings should be concise.
- Body text should remain readable at small sizes.
- Avoid oversized text except in hero or dashboard headline areas.
- Do not use negative letter spacing.

---

## 5. Spacing

The interface should feel calm and breathable.

Use:

- Consistent section gaps
- Generous card padding
- Clear spacing between form fields
- Balanced desktop layouts
- Compact but readable mobile spacing

Guidelines:

- Avoid crowded dashboards.
- Avoid nested cards where possible.
- Keep related information grouped.
- Use whitespace as a design tool.

---

## 6. Border Radius

Premium Dashboard v3 uses soft rounded corners.

Recommended:

- Small controls: `0.75rem`
- Buttons and inputs: `1rem`
- Cards: `1.25rem` to `1.5rem`
- Hero cards and major panels: `1.5rem` to `2rem`
- Circular icon containers: full radius

Avoid sharp square panels unless required by a data table.

---

## 7. Shadows

Use soft shadows to create depth without heaviness.

Recommended:

- Soft card shadows
- Subtle inner shadows for badges and icon containers
- Light elevation on hover
- No harsh black shadows

Shadows should feel warm and calm, not dramatic.

---

## 8. Buttons

Buttons should be clear, premium, and easy to understand.

Primary buttons:

- Coral background
- White text
- Rounded corners
- Soft shadow

Secondary buttons:

- White or cream background
- Navy text
- Soft border

Icon buttons:

- Rounded
- Accessible labels
- Consistent icon sizing

Guidelines:

- Use clear action labels.
- Avoid too many competing primary buttons on one screen.
- Disabled buttons should be visibly disabled.

---

## 9. Forms

Forms should feel modern and supportive.

Use:

- Clear labels
- Rounded inputs
- Soft borders
- Navy text
- Muted placeholder text
- Coral focus states
- Helpful validation messages

Guidelines:

- Group related fields.
- Keep forms readable on mobile.
- Do not hide required information.
- Avoid dense ERP-style form layouts.

---

## 10. Cards

Cards are the primary dashboard surface.

Use:

- White or warm cream backgrounds
- Rounded corners
- Soft shadows
- Clear headings
- Visual icons or accents
- Balanced padding

Cards should be used for:

- KPI metrics
- Dashboard widgets
- Repeated content
- Profile summaries
- Report panels

Avoid:

- Nested cards
- Dense tables inside decorative cards
- Excessive borders

---

## 11. Tables

Tables should be clean and readable.

Use:

- Soft borders
- Clear column headings
- Comfortable row height
- Subtle hover states
- Muted secondary text
- Responsive handling for smaller screens

Guidelines:

- Avoid dense table-first layouts when cards or summaries are more useful.
- Keep actions easy to find.
- Use empty states when there are no rows.

---

## 12. Charts

Charts should use the Premium Dashboard v3 palette.

Use:

- Sage for positive trends
- Coral for alerts or emphasis
- Warm yellow for secondary metrics
- Navy for labels
- Cream chart backgrounds

Guidelines:

- Keep charts simple.
- Use clear labels.
- Avoid bright or random colours.
- Avoid overly complex visualisations.

---

## 13. Sidebar

The sidebar should feel like a premium brand rail.

Use:

- Warm cream or soft white background
- Little London Play & Learn branding
- Coral active state
- Soft circular icon containers
- Role-aware navigation
- Scrollable navigation area
- Bottom operations card or brand illustration

Guidelines:

- Do not hide or rename navigation items differently between desktop and mobile.
- Preserve role-aware navigation rules.
- Keep active state obvious but elegant.

---

## 14. Header

The header should remain light, functional, and uncluttered.

Use:

- Soft cream/white background
- Search field
- Notification control where appropriate
- Language pill where appropriate
- User card
- Logout access

Guidelines:

- Do not include non-functional controls.
- Keep header height compact.
- Avoid crowding on tablet and mobile.

---

## 15. Icons

Use clean SVG icons.

Preferred:

- Lucide icons
- Simple inline SVGs for brand-specific illustrations
- Consistent stroke widths
- Soft circular icon backgrounds

Avoid:

- Low-resolution icons
- Emoji as functional UI icons
- Mixed icon styles
- Overly decorative icons in operational screens

---

## 16. Empty States

Empty states should be calm and helpful.

Use:

- Clear heading
- Short explanation
- Suggested next action where appropriate
- Soft icon or illustration

Tone:

- Friendly
- Reassuring
- Professional

---

## 17. Loading States

Loading states should preserve layout stability.

Use:

- Skeleton cards
- Subtle shimmer
- Reserved space for content
- Clear loading text where needed

Avoid:

- Layout jumping
- Full-page blank states when partial loading is possible
- Loud spinners

---

## 18. Animations

Animations should be subtle and premium.

Use:

- Gentle fade-ins
- Small upward motion
- Soft hover elevation
- Smooth chart drawing
- Calm transitions

Avoid:

- Flashy effects
- Bouncy cartoon movement
- Glitch effects
- Fast cuts
- Distracting animations

---

## 19. Responsive Design

All interfaces must support:

- Desktop
- Tablet
- Mobile

Guidelines:

- Desktop layouts may use multi-column dashboards.
- Tablet layouts should stack cleanly.
- Mobile layouts should prioritise readability and actions.
- No horizontal overflow.
- No clipped text.
- Navigation must remain usable.

---

## 20. Accessibility

Accessibility is part of the design system.

Requirements:

- Maintain readable contrast.
- Use semantic headings.
- Provide accessible labels for icon-only buttons.
- Preserve keyboard focus states.
- Avoid relying on colour alone.
- Keep text readable on all screen sizes.
- Ensure forms have labels and validation messages.

---

## 21. Future UI Principles

Future UI work must extend the approved Premium Boutique Dashboard v3 language.

Principles:

- Do not invent a new visual system.
- Keep the Premium Login Experience as the brand reference.
- Keep Premium Dashboard v3 as the dashboard and management reference.
- Use consistent card, button, form, and table patterns.
- Preserve role-aware navigation.
- Prioritise clarity over decoration.
- Keep workflows calm and easy to understand.
- Build premium UI that feels worth trusting.

Every new module should look like it belongs naturally inside the Little London Management System.
