# Little London Management System

# UI_UX_GUIDELINES.md

Version: 1.0

---

# Vision

The Little London Management System should feel like a premium international education platform.

It should not resemble an old-fashioned school ERP or accounting software.

Every interaction should be modern, intuitive, warm, and professional.

The interface should create trust with parents, improve teacher productivity, and give management complete visibility over the business.

The application should be designed with scalability in mind and support future expansion to multiple branches.

---

# Design Inspiration

The UI and UX should take inspiration from:

* Stripe Dashboard
* Linear
* Notion
* Framer
* Vercel Dashboard
* Apple
* Airbnb
* Google Material Design (minimal use)
* shadcn/ui component philosophy

Do not copy these products. Instead, adopt their simplicity, consistency, spacing, and premium feel.

---

# Brand Personality

Little London is:

* Friendly
* Modern
* Educational
* Trustworthy
* Premium
* Playful without being childish
* Elegant
* Calm
* Welcoming

The interface should appeal to both parents and business users.

---

# Visual Design Language

The design should use:

* Large whitespace
* Soft rounded corners
* Premium typography
* Clean layouts
* Minimal borders
* Subtle shadows
* Soft gradients
* Calm color palette
* Smooth animations

Avoid visual clutter.

# Typography

Primary Font:
Inter

Fallback:
System Sans Serif

Heading Weight:
600–700

Body Weight:
400–500

Minimum font size:
14px

Never use fonts smaller than 14px.

Use clear hierarchy for headings, subtitles, labels, and helper text.

---

# Layout

The application should have:

* Fixed left sidebar
* Sticky top navigation
* Responsive content area
* Maximum readable content width
* Consistent spacing throughout
* Desktop-first layout
* Mobile-friendly adaptation

---

# Sidebar

The sidebar should include:

* Little London logo
* Role-specific navigation
* Icons with labels
* Collapsible mode
* Active page indicator
* Quick search
* User profile at the bottom

Navigation should differ for Management, Teachers, and Parents.

---

# Dashboard Experience

The dashboard should immediately answer:

* What needs attention today?
* Which classes are running?
* Who is absent?
* Which invoices are overdue?
* What events are coming?
* What recent registrations occurred?
* Are there teacher remarks awaiting approval?

Use information cards, charts, and quick actions.

---

# Cards

Cards should include:

* Rounded corners (16px)
* Light border
* Soft shadow
* Icon
* Title
* Supporting information
* Optional actions

Avoid overcrowding.

---

# Tables

All tables should support:

* Search
* Sorting
* Filtering
* Pagination
* Sticky headers
* Bulk actions
* Export to CSV
* Responsive layout
* Empty states
* Loading skeletons

---

# Forms

Forms should be easy to complete.

Use:

* Large input fields
* Clear labels
* Inline validation
* Helpful placeholders
* Multi-step registration
* Progress indicators
* Auto-save where appropriate

Group related fields together.

---

# Buttons

Button styles:

* Primary
* Secondary
* Outline
* Ghost
* Danger
* Success

Every button should have:

* Hover state
* Focus state
* Disabled state
* Loading state

---

# Icons

Use Lucide Icons consistently.

Icons should improve recognition without replacing text labels.

---

# Animations

Use Framer Motion.

Animations should be:

* Fast
* Smooth
* Purposeful
* Subtle

Avoid excessive movement.

---

# Notifications

Use toast notifications for:

* Success
* Warning
* Error
* Information

Messages should be concise and user-friendly.

---

# Empty States

Every module should include:

* Friendly illustration
* Helpful explanation
* Clear call-to-action

Example:

"No students have been registered yet."

Action:

"Register Student"

---

# Loading States

Use skeleton loaders for:

* Tables
* Cards
* Dashboard widgets
* Forms

Avoid layout shifts.

---

# Accessibility

Support:

* Keyboard navigation
* ARIA labels
* Visible focus indicators
* High color contrast
* Screen readers
* Responsive font sizes

---

# Parent Portal

The Parent Portal should be extremely simple.

Parents should be able to:

* View their children
* Check attendance
* See class schedules
* Read approved teacher remarks
* Download invoices
* View payment history
* Read announcements
* Update contact details

Use larger buttons and minimal navigation.

---

# Teacher Portal

Teachers should have quick access to:

* Today's classes
* Student lists
* Attendance
* Homework
* Class notes
* Remarks
* Announcements
* Weekly timetable

The interface should reduce repetitive tasks.

---

# Management Portal

Management should have access to:

* Analytics dashboard
* Student management
* Parent management
* Teacher management
* Courses
* Workshops
* Nursery
* Birthday events
* Attendance
* Payments
* Invoices
* Reports
* Settings

Everything should be searchable, filterable, and exportable.

---

# Mobile Experience

Parents are expected to use the system on mobile devices.

The mobile interface should include:

* Bottom navigation
* Large touch targets
* Simplified layouts
* Fast loading
* Responsive tables
* Mobile-friendly forms

Management and teacher interfaces should remain fully usable on tablets.

---

# Dark Mode

Support both Light Mode and Dark Mode.

Remember the user's preference.

Ensure charts, tables, and cards remain readable in both themes.

---

# Overall Goal

Every screen should look like it belongs to a premium SaaS product.

Users should immediately feel that Little London is a modern international education company.

The experience should emphasize simplicity, trust, efficiency, and elegance.
